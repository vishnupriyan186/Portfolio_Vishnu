## 3. ABOUT ME

In a world where digital interfaces are the new frontier, I operate at the intersection of human-centered design and cybersecurity engineering. My journey began with a fascination for how people interact with technology—the psychology behind every click, the emotion behind every interaction. This curiosity led me to master the art of UI/UX design, creating interfaces that don't just look beautiful, but feel intuitive and purposeful.

Parallel to this creative pursuit, I developed a deep understanding of the vulnerabilities that exist beneath the surface. As a cybersecurity engineer, I analyze systems, identify weaknesses, and build defenses that stand against evolving threats. I've developed intrusion detection systems, digital forensics tools, and security frameworks that protect critical infrastructure.

What sets my approach apart is the synthesis of these disciplines. When I design, I think like a security engineer—considering attack vectors, data privacy, and secure architecture. When I secure systems, I think like a designer—ensuring security measures enhance rather than hinder user experience. This dual perspective allows me to create solutions that are both beautiful and bulletproof, user-friendly and secure.

I believe the best technology is invisible—it works so seamlessly that users never think about it, while security measures operate silently in the background. My work embodies this philosophy: elegant interfaces that users trust, protected by robust security that they never see.

**Problem:** The existing app had a 23% cart abandonment rate and low user engagement. Navigation was confusing, checkout process was lengthy, and visual hierarchy was poor.

**Approach:** Conducted user research through surveys and interviews, created user personas, mapped customer journeys, and identified pain points. Designed new information architecture with simplified navigation. Implemented a streamlined checkout process reducing steps from 5 to 2. Created a cohesive design system with improved typography, spacing, and color contrast.

**Tools Used:** Figma, Adobe XD, Principle (prototyping), UserTesting, Analytics tools

**Results:** 
- 40% reduction in cart abandonment

- 35% improvement in average session duration

### Cyber Projects

#### Project 1: Network Intrusion Detection System (IDS)
**Overview:** Developed a custom IDS solution capable of real-time network traffic analysis, anomaly detection, and automated threat response.

**Problem:** Existing IDS solutions were either too expensive for small organizations or lacked customization for specific network environments. False positive rates were high, and integration with existing infrastructure was complex.

**Approach:** Analyzed network traffic patterns and identified key attack signatures. Implemented signature-based detection combined with anomaly-based detection using machine learning algorithms. Created a modular architecture allowing easy rule updates. Built a web-based dashboard for monitoring and alerting.

**Tools Used:** Python, Scapy, Snort (for signature rules), Machine Learning libraries (scikit-learn), Flask (for dashboard), SQLite (for logging)

**Results:**
- 94% detection accuracy for known attack patterns
- 78% reduction in false positives compared to baseline
- Real-time alerting with <2 second latency
- Successfully deployed in production environment

---

#### Project 2: Automated Brute Force Detection & Prevention Tool
**Overview:** Created a security tool that detects and mitigates brute force attacks in real-time, with adaptive response mechanisms.

**Problem:** Organizations needed better protection against credential stuffing and brute force attacks. Existing solutions were reactive rather than proactive, and lacked intelligent rate limiting.

**Approach:** Implemented connection tracking and pattern recognition algorithms. Created adaptive rate limiting that adjusts based on attack severity. Developed automated IP blocking with whitelist management. Built integration with existing authentication systems.

**Tools Used:** Python, iptables (for IP blocking), Redis (for rate limiting), Log analysis tools

**Results:**
- 100% detection rate for brute force attempts
- Automated blocking reduced manual intervention by 85%
- Zero successful brute force attacks during testing period
- Reduced server load during attack attempts by 60%

---

#### Project 3: Digital Forensics & Incident Response (DFIR) Tool
**Overview:** Developed a comprehensive DFIR toolkit for rapid incident response, evidence collection, and forensic analysis.

**Problem:** Incident response teams needed a unified tool for collecting and analyzing forensic data across multiple systems. Existing tools were fragmented and required extensive manual work.

**Approach:** Created modular toolset for different forensic tasks: memory analysis, disk imaging, log analysis, and timeline reconstruction. Implemented automated evidence collection scripts. Built correlation engine to connect events across multiple sources. Designed secure evidence storage with chain of custody tracking.

**Tools Used:** Python, Volatility (memory forensics), Autopsy (disk analysis), Elasticsearch (for log correlation), Cryptography libraries

**Results:**
- 70% reduction in incident response time
- Automated evidence collection saved 15+ hours per incident
- Improved evidence integrity with automated chain of custody
- Successfully used in 12+ real-world incident investigations

---

#### Project 4: Reverse Shell Detection & Analysis Framework
**Overview:** Built a framework for detecting, analyzing, and responding to reverse shell connections in network environments.

**Problem:** Reverse shells are difficult to detect as they use legitimate protocols. Security teams needed better visibility into suspicious outbound connections and command execution patterns.

**Approach:** Implemented network traffic analysis to detect suspicious outbound connections. Created behavioral analysis to identify command execution patterns. Developed correlation engine linking network events to system events. Built automated response mechanisms for detected shells.

**Tools Used:** Python, Network packet analysis libraries, System monitoring tools, Machine learning for pattern recognition

**Results:**
- 89% detection rate for reverse shell connections
- Average detection time: 3.2 minutes from connection establishment
- Automated response prevented data exfiltration in test scenarios
- Reduced false positives through behavioral analysis

---

#### Project 5: OSINT (Open Source Intelligence) Automation Framework
**Overview:** Developed an automated OSINT collection and analysis framework for security research and threat intelligence gathering.

**Problem:** Manual OSINT collection is time-consuming and inconsistent. Security researchers needed a tool to automate data collection from multiple sources and correlate findings.

**Approach:** Integrated APIs from multiple OSINT sources (social media, domain registrars, certificate transparency logs, etc.). Created data normalization and correlation engine. Built visualization tools for relationship mapping. Implemented automated reporting with actionable intelligence.

**Tools Used:** Python, Various OSINT APIs, NetworkX (for relationship graphs), Data visualization libraries, Web scraping tools (ethical, with rate limiting)

**Results:**
- 90% reduction in OSINT collection time
- Automated correlation identified 15+ previously unknown connections
- Generated actionable intelligence reports in minutes vs. hours
- Successfully used in multiple security research projects
